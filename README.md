# Whisk: Reel to Recipe

A dependency-free web app that turns Instagram Reels and YouTube food videos into kitchen-ready recipes with a free local AI.

## GitHub-friendly release

This public-ready copy does not commit large video-tool binaries. GitHub blocks files over 25 MB, so `tools/ffmpeg` and `tools/yt-dlp_macos` are intentionally ignored and downloaded by setup instead.

The source repo stays small; contributors run one setup command after cloning.

## Free local AI setup

Whisk uses [Ollama](https://ollama.com/download) and open local Gemma vision models. There is no API key, subscription, or per-recipe charge. `Fast local` uses `gemma3:4b` by default; `Deep review` keeps `gemma3:12b` available for complex clips.

1. [Download Ollama for macOS](https://ollama.com/download/mac), move it to Applications, and open it.

2. Download the default Fast local vision model once:

```bash
ollama pull gemma3:4b
```

3. Optionally download the slower Deep review model:

```bash
ollama pull gemma3:12b
```

4. Download the local video tools after cloning:

```bash
npm run setup:tools
```

5. Start Whisk:

```bash
npm start
```

Then open [http://localhost:4173](http://localhost:4173).

The UI also has a built-in sample recipe, so the full interaction can be previewed before local AI setup is complete.

## Publishing to GitHub

Commit this folder as your public app. Do not commit files generated by `npm run setup:tools`; `.gitignore` excludes them:

- `tools/ffmpeg`
- `tools/yt-dlp_macos`

If you upload files through GitHub's web UI instead of committing with Git, run the upload before `npm run setup:tools`, or manually leave those two downloaded binaries out.

## How analysis works

- Paste an Instagram Reel or YouTube URL.
- Whisk downloads a compact public visual stream into a temporary folder, samples ten frames locally, and deletes the temporary media after analysis.
- For YouTube links, Whisk also retrieves creator captions or auto-generated captions when the video exposes them.
- Attach a saved video clip only when a platform blocks automatic download. The browser samples ten fallback frames for the same local vision model.
- Choose `Fast local` for the normal evidence-preserving path or `Deep review` when a complicated clip justifies the larger model and longer wait.
- Optionally add notes for details that captions may miss, such as ingredients shown only on screen.
- The server fetches public URL metadata where available, extracts a conservative evidence report, then asks the model running on your computer for a grounded structured recipe.

Whisk overlaps caption retrieval with video download, keeps the local model warm between requests, and caches a small number of recent URL analyses in memory for quicker repeat visits.

## Privacy and platform limitation

The AI analysis stays on your computer. Fetching a public title, caption track, or video stream for a pasted link still contacts YouTube or Instagram. Temporary downloaded media is deleted after its frames are sampled.

Instagram and YouTube can prevent automated tools from fetching actual video bytes from some URLs. When that happens, attach a saved clip so the local model still has visual evidence. Whisk refuses requests without visual frames instead of using captions as a shortcut and generating a polished guess.

Whisk downloads [`yt-dlp`](https://github.com/yt-dlp/yt-dlp) and a macOS [`FFmpeg`](https://ffmpeg.org/download.html) build for free platform-video download and local frame extraction. Attribution details are in [`tools/README.md`](tools/README.md).
